# Security Policy

## Supported Versions

The following versions of our React Native application currently receive security updates and patches.

| Version | Supported          |
| ------- | ------------------ |
| 5.1.x   | ✅ Supported       |
| 5.0.x   | ❌ Not Supported   |
| 4.0.x   | ✅ Supported       |
| < 4.0   | ❌ Not Supported   |

Only supported versions receive:
- Security patches  
- Critical vulnerability fixes  
- Platform-related security updates (Android/iOS)  
- Dependency upgrades for security

---

## Reporting a Vulnerability

We take security seriously. If you discover a vulnerability in our React Native app, please report it responsibly.

### How to Report

Email us at: **security@[yourdomain].com**  
(Replace with your actual security contact email.)

Include, if possible:
- Description of the vulnerability  
- Steps to reproduce  
- Affected module or screen  
- Expected vs. actual behavior  
- Screenshots, logs, or evidence  
- Proof of concept (optional)

### What to Expect

- **Acknowledgment:** We will respond within **48 hours**.  
- **Assessment:** Our security team will validate and categorize the report.  
- **Updates:** You will receive a status update every **5 business days**.  
- **Resolution:** If confirmed, the issue will be patched in the next security release and you will be notified.

If the issue is declined (e.g., non-exploitable or expected behavior), we will provide a clear explanation.

---

## Scope

The following areas are in scope:
- React Native application code (JS/TS)  
- Native modules and bridges  
- API integrations and data handling  
- Authentication, encryption, token handling  
- Secure storage and permission-related functionality  
- OTA update systems (CodePush, Expo Updates)

Out of scope:
- Social engineering  
- Third-party service outages  
- Issues found in unsupported versions

---

## Responsible Disclosure

Please do **not publish** any discovered vulnerabilities publicly until a fix has been released.  
We strongly encourage coordinated disclosure to protect all users.
